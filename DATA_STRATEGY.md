# 실제 데이터 수집 전략

## 현재 상황
- **샘플 데이터**: 7개 병원, 7명 의사 (완전히 가짜)
- **문제점**: 실제 사용자가 활용할 수 없음

## 실제 데이터 소스 분석

### 1. 공공데이터포털 (data.go.kr) ⭐ 추천
**장점:**
- 정부 공식 데이터, 신뢰도 높음
- 무료 API 제공
- 법적 문제 없음

**주요 API:**

#### A. 건강보험심사평가원 - 병원정보서비스
- **API명**: 병원 및 약국 정보조회 서비스
- **제공 정보**:
  - 요양기관명 (병원명)
  - 주소, 전화번호
  - 병원 종별 (상급종합병원, 종합병원, 병원, 의원 등)
  - 진료과목
  - 병상 수
  - 응급실 운영 여부
- **데이터 건수**: 약 9만개 의료기관
- **URL**: https://www.data.go.kr/data/15051059/openapi.do

#### B. 국민건강보험공단 - 요양기관 정보
- **제공 정보**:
  - 기관 기본정보
  - 진료과목
  - 의료장비 보유 현황
- **URL**: https://www.data.go.kr/data/15007431/openapi.do

#### C. 서울시 열린데이터광장 - 병원 위치정보
- **제공 정보**:
  - 병원 좌표 (위도, 경도)
  - 상세 주소
- **URL**: https://data.seoul.go.kr/

**API 발급 절차:**
1. 공공데이터포털 회원가입
2. 원하는 API 활용신청
3. 인증키 발급 (즉시 또는 1-2일 소요)
4. API 호출 시 인증키 사용

**사용 제약:**
- 일일 트래픽 제한 (보통 1,000~10,000 건)
- 상업적 이용 시 별도 승인 필요할 수 있음

### 2. 의사 정보 - 현실적 한계 ⚠️

**문제점:**
- **개인정보보호법**: 의사 개인정보 (이름, 연락처, 이메일) 공개 불가
- **의료법**: 의료인 정보 무단 수집 금지

**사용 가능한 데이터:**
- ✅ 병원별 진료과 존재 여부
- ✅ 전문의 수 (통계)
- ❌ 의사 개인 이름, 연락처
- ❌ 의사 학력, 경력 (병원 홈페이지 공개 정보 제외)

**대안:**

#### A. 병원 자체 등록 시스템
- 병원이 직접 소속 의사 정보 입력
- 의사 동의 하에 정보 공개
- 병원 관리자 권한 필요

#### B. 병원 홈페이지 공개 정보 활용
- 대형 병원들은 홈페이지에 의료진 소개 페이지 운영
- 예: 서울대병원, 삼성서울병원 등
- **주의**: 크롤링 시 robots.txt 준수 필수
- **법적 리스크**: 무단 크롤링 시 저작권법 위반 가능

#### C. 의료진 정보 제공 플랫폼 연동
- 굿닥, 강남언니 등 의료 플랫폼 API (유료)
- 병원 예약 시스템 연동

### 3. 추천 솔루션 - 하이브리드 방식

#### 단계별 구현 전략

**Phase 1: 공공 병원 데이터 (즉시 구현 가능)**
```
공공데이터포털 API 연동
  ↓
실제 병원 9만개 데이터 수집
  ↓
DB 저장 및 정기 업데이트 (월 1회)
```

**Phase 2: 병원 관리자 등록 시스템**
```
병원 회원가입 및 인증
  ↓
병원이 직접 의사 정보 등록
  ↓
의사 프로필 승인 시스템
```

**Phase 3: 공개 정보 수집 (선택)**
```
대형 병원 공식 홈페이지
  ↓
공개된 의료진 정보만 수집
  ↓
병원 동의 하에 데이터 활용
```

## 법적 고려사항

### 준수해야 할 법률
1. **개인정보보호법**
   - 민감정보(의료정보) 처리 시 별도 동의 필요
   - 개인정보 처리방침 명시

2. **의료법**
   - 의료광고 심의 필요 여부 확인
   - 허위 의료정보 게시 금지

3. **저작권법**
   - 크롤링 시 데이터베이스권 침해 주의
   - robots.txt 준수

### 권장 사항
- 병원/의사 정보 게시 전 동의 획득
- 이용약관에 데이터 수집/활용 명시
- 정보 삭제 요청 프로세스 마련

## 비용 분석

### 무료 옵션
- 공공데이터포털 API: 무료 (트래픽 제한 있음)
- 자체 데이터 수집: 개발 인력만 필요

### 유료 옵션
- 의료 플랫폼 API: 월 50만원~
- 크롤링 대행 서비스: 건당 과금
- 클라우드 DB: 월 5만원~ (데이터양에 따라)

## 즉시 적용 가능한 솔루션

**지금 바로 구현:**
1. 공공데이터포털 API로 실제 병원 데이터 수집
2. 의사 정보는 병원 자체 등록 방식으로 전환
3. 샘플 의사 데이터에 "샘플" 표시 추가

**다음 단계:**
