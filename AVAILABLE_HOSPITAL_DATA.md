## 🔌 공공 API를 통한 데이터 수집

### 1. 건강보험심사평가원 병원정보서비스 API

**API 제공 기관:** 건강보험심사평가원 (HIRA)
**데이터 포털:** https://www.data.go.kr/data/15051059/openapi.do

#### 제공 정보

| 카테고리 | 제공 내용 |
|---------|----------|
| **기본 정보** | 병원명, 종별코드, 주소, 전화번호, 홈페이지 |
| **위치 정보** | 좌표(위도/경도), 시도, 시군구, 읍면동 |
| **진료 정보** | 진료과목코드, 응급실 운영 여부 |
| **인력 현황** | 의사 수, 전문의 수, 레지던트 수, 인턴 수 |
| **시설 정보** | 병상 수, 의료장비 현황 |

#### API 엔드포인트


## 📡 데이터 수집 가이드

### API 인증키 발급 방법

1. **공공데이터포털 회원가입**
   - https://www.data.go.kr 접속
   - 회원가입 및 로그인

2. **API 활용신청**
   - "건강보험심사평가원 병원정보서비스" 검색
   - "활용신청" 버튼 클릭
   - 활용 목적 작성 (예: 병원·의사 정보 제공 웹서비스)

3. **승인 대기**
   - 승인까지 약 1~2시간 소요
   - 이메일로 승인 알림 수신

4. **인증키 확인**
   - 마이페이지 > 오픈API > 인증키 발급현황
   - 일반 인증키(디코딩) 사용 권장

### 데이터 수집 스크립트 사용

프로젝트에 포함된 `backend/src/scripts/fetchRealData.js`를 사용하여 자동으로 병원 데이터를 수집할 수 있습니다.

#### 사용 방법

1. **환경 변수 설정**
   ```bash
   # backend/.env 파일 생성
   PUBLIC_DATA_API_KEY=발급받은_API_키
   PORT=5000
   ```

2. **스크립트 실행**
   ```bash
   cd backend
   node src/scripts/fetchRealData.js
   ```

3. **수집 대상 지역**
   - 서울, 부산, 대구, 인천, 광주, 대전
   - 각 지역별 100개씩 자동 수집

4. **수집 속도 제어**
   - API 호출 간 1초 대기 (Rate Limiting)
   - 너무 빠른 요청은 API 차단될 수 있음

#### 스크립트 커스터마이징

```javascript
// 수집 대상 지역 변경
const TARGET_REGIONS = ['서울', '부산', '대구', '인천', '광주', '대전'];

// 페이지당 수집 개수 변경 (최대 100)
const numOfRows = 100;

// 특정 병원 종별만 수집
const clCd = '11'; // 상급종합병원만
```

## ⚖️ 법적 고려사항

### 공공 데이터 이용 시 준수사항

1. **출처 명시 의무**
   - 데이터 출처를 명확히 표시해야 함
   - 예: "본 데이터는 건강보험심사평가원에서 제공한 정보입니다."

2. **이용 허가 범위**
   - 공공데이터포털 이용약관 준수
   - 상업적 이용 가능 (별도 제한 없음)
   - 재배포 시 출처 명시 필수

3. **개인정보보호**
   - API는 병원 기본 정보만 제공 (개인정보 없음)
   - 의사 개인 정보는 포함되지 않음
   - 의사 정보는 병원 직접 등록 방식 사용

### 본 시스템의 법적 대응

✅ **Terms.jsx** - 이용약관에 데이터 출처 명시
✅ **Privacy.jsx** - 개인정보처리방침 게시
✅ **Footer.jsx** - 모든 페이지에 데이터 출처 표시
✅ **병원 자체 등록 시스템** - 의사 정보는 병원이 직접 관리

### Phase 1: 병원 정보 수집

   - 네트워크 오류 시 해당 지역만 재시도

2. **종별 우선순위**
   ```
   1순위: 상급종합병원 (47개)
   2순위: 종합병원 (331개)
   3순위: 병원 (1,403개)
   4순위: 의원 (35,717개)
   ```

3. **캐싱 전략**
   - 병원 기본 정보는 변경 빈도가 낮음
   - 로컬 데이터베이스에 저장 후 주기적 업데이트
   - API 호출 최소화로 속도 향상

### API 호출 최적화

```javascript
// ✅ Good: 지역별 일괄 조회
await getHospitalList({ sidoCd: '110000', numOfRows: 100 });

// ❌ Bad: 병원 하나씩 상세 조회
for (const hospital of hospitals) {
  await getHospitalDetail(hospital.id); // 너무 많은 API 호출
}
```

### 에러 처리

```javascript
try {
  const result = await publicDataAPI.getHospitalList(params);
  if (result.success) {
    // 데이터 저장
  }
} catch (error) {
  if (error.code === 'RATE_LIMIT') {
    // 1초 대기 후 재시도
    await delay(1000);
    retry();
  } else if (error.code === 'INVALID_API_KEY') {
    // API 키 확인 필요
    console.error('API 키를 확인해주세요');
  }
}
```
